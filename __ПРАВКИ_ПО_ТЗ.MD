# Тестовое задание для кандидата на должность full-stack JavaScript

## Описание

Для демонстрации своих навыков кандидат на должность full-stack JavaScript разработчика должен реализовать приложение Todo manager. Главная цель тестового задания – проверить, имеет ли кандидат навыки в технологиях. Плановые трудозатраты для решения этого задания – 8 часов опытного разработчика. Полнота и качество реализации являются более важным фактором, чем реализация большего количества функций. Вы можете задавать любые вопросы о требованиях сроках и используемых технологиях. Вы можете использовать любые шаблоны (заготовки, генераторы кода, т.д.), если результат покрывает все требования.

## Высокоуровневое описание
Приложение Todo manager помогает пользователю отслеживать прогресс выполнения задач. Например, когда вы разрабатываете приложение, вы создаете задачи, вроде UI Design, HTML Markup, Front-end, Back-end, Database design и ведете их статус. Приложение проектируется на одного пользователя, поэтому никаких взаимодействий между пользователями не будет.

## Сценарии использования
### Просмотр списка todo
Пользователь имеет возможность посмотреть список всех своих todo
### Добавить новую todo
Пользователь имеет возможность добавить новую todo в список. В todo есть следующие поля: 
- Заголовок Todo (до 50 символов). 
- Приоритет Todo (выбирается из списка «High», «Middle», «Low»). 
- Состояние Todo (изначально «New», так же могут быть статусы «Active», «Resolved», «Closed»). 
- Крайний срок (не раньше сегодняшнего дня).
- ### Удалить todo
- Пользователь имеет возможность удалить любую todo. Приложение должно запросить подтверждение перед удалением.
### Отредактировать todo
Пользователь имеет возможность отредактировать любое поле любой todo.
### Изменить порядок в списке todo
Пользователь имеет возможность изменить порядок todo в списке.

## Нефункциональные требования
### Общие требования
• У приложения должен быть приятный интерфейс 
• Приложение должно быть реализовано как связка REST API для управления данными и одностраничное приложение (Single Page Application) для пользовательского интерфейса 
• Для проверки необходимо предоставить ссылку на публичный репозиторий с исходным кодом и ссылку на опубликованное приложение.
### Требования к API
• REST API должен быть реализован как Node JS приложение с использование популярных библиотек для обработки запросов, например, express или koa 
• Необходимо реализовать серверную валидацию
### Требования к SPA
- SPA должно быть реализовано с помощью React JS 
- Необходимо реализовать клиентскую валидацию 
- Необходимо использовать CSS препроцессоры (SASS or LESS) 
- Необходимо использовать Webpack 2 (или выше) для сборки JS и CSS. На выходе билда должны быть следующие файлы: 
	- bundle.css – все стили 
	- vendor.js – JS библиотек 
	- bundle .js – JS приложения
### Требования к данным
Данные должны сохраняться в базу данных, чтобы при открытии приложения с другого устройства пользователь увидел ранее сделанные изменения.
### Дополнительные требования
Реализация этих требования не обязательна, но будет плюсом при проверке. 
- Использовать возможности ES6 
- Использовать mongoDB для хранения данных 
- Реализовать server side rendering 
- Подключить ESLint и сделать код валидным для ESLint config 
- Использовать SVG формат для иконок
